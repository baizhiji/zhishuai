## 二、核心架构
### APK 端（移动应用）
为了快速将智枢 AI Web 端上线试用，有些在 APK 上开发比较复杂、耗时的功能就放到Web 端（管理后台）去配置，APK 端只做为功能基本使用/查看端。
### Web 端（管理后台）
APK 上开发比较复杂、耗时的功能就放到网页端后台，各种 APK 功能使用需要的配置都在Web 端（管理后台）完成（如：配置其他 API 服务商、客服知识库、批量剪辑……）。Web 端里智枢 AI APK 所有功能都能使用，还要有数据大盘（数据大盘根据不同权限的使用者看到内容不一样）。
## 三、账号体系
### 三级账号体系（注册/登录用手机号码 + 密码，初始密码设定为123456，客户可自行修改）
| 角色 | 说明 | 权限 |
| admin | 开发者总后台 | 最高权限，可管理所有账号和配置，开通区域代理帐号，开关区域代理权限，【名称配置（贴牌定制）、LOGO 配置（贴牌定制）仅开发都总后台可配置】只做为管理总台为设置、查看下级代理商、客户可以使用什么功能进行设置、推荐数据、使用数据等，不需要具体的功能。
| agent | 区域代理 | 管理终端客户，为开通客户账号、功能开关、客户可以使用什么功能进行设置、推荐数据、使用数据等，不需要具体的功能。
| customer | 终端客户 | 使用 APP 功能，后台可以使用所有功能，所有功能使用时所需要的技能实现渠道（如添加自媒体平台账号、电商平台账号、招聘网站账号、添加 API 服务商等需要设置的内容）不可见功能开关。
### 功能系统
整个"智枢 AI APK"项目是打造成一个"商品"，里面的功能（AI智能内容生成、推荐分享功能为基础功能，所有用户全部都开通），其他功能使用看用户使用需求开通（开发者总后台最高权限，区域代理开通终端客户功能，要在用户管理/设置里有功能开关），所以客户所有（不管 APK 端、Web 端）使用功能都是可以关闭的。由开发者总后台和区域代理操作。

## 六、页面结构
设计一个**SaaS多租户系统**，包括 **APK 端（移动应用）** 和 **Web 端（管理后台）**，核心模块为自媒体、招聘、获客、推荐分享、转介绍，AI 能力嵌入各业务板块并自动保存内容到素材库，API 服务商配置由终端客户自行完成，贴牌定制仅限 Admin。电商等模块为预留扩展。
下面我为您提供一份**清晰、可直接落地的页面结构设计方案**，以及配套的**设计话术**（用于与团队或客户沟通）。我会按照 **APK 端** 和 **Web 端（分角色）** 分别说明。
## 整体设计原则（先对齐团队认知）
> **设计话术**："智枢 AI 的页面结构必须遵循四个原则：
> **1. 商品化** – 功能开关控制菜单显隐，未开通功能直接消失。
> **2. 角色隔离** – Admin/Agent/Customer 导航完全独立，贴牌仅 Admin 可见。
> **3. 移动轻量，后台全面** – APK 只做高频使用和查看，所有复杂配置（API密钥、知识库、批量参数）放在 Customer 的 Web 后台。
> **4. AI 能力嵌入业务** – 不设独立 AI 板块，生成内容自动进入素材库，可在各业务中复用。"
## （一）、APK 端（移动应用）页面结构
**底部导航栏：3 个 Tab – 首页、AI 创作、我的**
### 1. 首页 Tab（功能矩阵 + 数据摘要）
| 区块 | 内容 | 说明 |
|------|------|------|
| 顶部栏 | 欢迎语（如"下午好，xx科技"）、系统公告图标 | 租户名称由 Admin 配置（贴牌），客户不可改 |

| 快捷操作区 | 最近使用的 3 个功能（例如"批量剪辑""智能文案""发布记录"） | 基于用户行为动态推荐 |

> **话术**："AI 创作 Tab 聚合所有生成能力，生成内容自动进入素材库，可在自媒体发布、招聘职位描述等场景中复用。电商详情页生成作为 AI 能力先占位，不开发完整电商流程。"

## （二）、Web 端（管理后台）页面结构
采用 **左侧导航 + 顶部栏 + 内容区** 布局。三个角色导航完全不同。
### 角色 A：Admin（开发者总后台）
Admin **不操作具体业务**，只做平台管控。**贴牌配置仅在此角色出现**。
| 一级菜单 | 说明 | 关键操作 |
|---------|------|---------|
| 租户管理 | 查看所有 Agent 和 Customer 列表 | 强制开关功能、调整套餐、重置密码、冻结/解冻 |
| 代理商管理 | 创建/冻结区域代理账号 | 设置代理分成比例、可售卖功能范围 |
| 功能开关总控 | 全局功能字典（自媒体、招聘、获客、推荐分享、转介绍） | 设置默认对所有租户开放/关闭 |
| **贴牌配置** | 全局默认 APP 名称、LOGO（上传图片）、默认主题色 | **仅 Admin 可配置**，Agent 和 Customer 无此菜单 |
| 数据大盘 | 全平台统计：总客户数、活跃度、功能使用排行、API 调用量 | 支持下钻到代理商，导出报表 |
| 系统配置 | 热更新版本管理、全局公告、帮助文档 | 不包含 API 服务商配置（由 Customer 自己配置） |
| 操作日志 | 记录所有 Admin 和 Agent 的关键操作 | 审计追溯 |
> **话术**："Admin 后台是平台方控制中心，贴牌配置只在这里出现，确保品牌形象统一。API 服务商配置下放给 Customer，因为每个客户使用自己的第三方账号，Admin 不接触客户密钥。"
### 角色 B：Agent（区域代理后台）
Agent 管理名下终端客户，**没有贴牌配置，没有 API 配置**。
| 一级菜单 | 说明 | 关键操作 |
|---------|------|---------|
| 客户管理 | 查看名下所有 Customer | 新增客户、重置密码、设置套餐到期时间、冻结/解冻 |
| 功能开关管理 | 针对每个客户，表格形式列出所有功能模块 | 开关勾选后客户即刻可见，支持批量操作 |
| 推荐数据看板 | 查看名下所有客户的推荐总人数、活跃用户数、产生的订单价值 | 用于代理商业绩分析 |
| 使用数据报表 | 查看名下客户的功能调用量（API 次数、发布量、获客数、沟通数） | 可导出，用于客户活跃度分析 |
| 工单/申请处理 | 接收客户的功能开通申请 | 审批通过或拒绝，填写备注 |
> **话术**："Agent 后台专注客户管理和功能授权。贴牌和 API 配置都不属于代理职责，保持角色职责清晰。"
### 角色 C：Customer（终端客户后台）—— 最复杂

## （三）、关键设计话术汇总（直接用于团队沟通）
### 1. 向研发解释贴牌权限的收缩
> "最新需求明确：贴牌定制（名称/LOGO）仅限开发者总后台（Admin）配置，代理商和终端客户都不能修改。所以前端只有 Admin 后台才有'贴牌配置'页面，APK 端和 Customer 后台不再提供任何修改名称/LOGO 的入口。这样保证平台整体品牌形象的一致性，避免代理商随意篡改。"
### 2. 关于 AI 内容自动保存到素材库
> "所有 AI 生成的内容（文本、图片、视频、数字人视频、电商详情页）在生成后必须自动保存到'素材库'中，并在对应的业务板块（如自媒体发布中心）提供'从素材库选择'的入口。素材库应支持分类、搜索、标记使用状态（是否已发布过），以遵守'每个素材只能使用一次'的需求。API 设计上，生成接口应同时返回素材 ID，方便前端跳转。"
### 3. API 服务商配置的下放
> "需求明确：API 服务商配置由终端客户（Customer）在 Web 端完成。Admin 和 Agent 不需要配置，因为他们不使用具体功能。"
