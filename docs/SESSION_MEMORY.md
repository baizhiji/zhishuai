## 🔄 自动记忆机制

### 1. 会话记忆 (OpenClaw)
- **位置**: OpenClaw 向量数据库
- **内容**: 对话历史、项目上下文
- **持久化**: 跨会话保留

### 2. 文件记忆 (Git)
- **位置**: `/workspace/zhishuai/.git/`
- **内容**: 所有代码变更
- **持久化**: 永久保留

### 3. 文档记忆
| 文档 | 内容 | 更新频率 |
|------|------|---------|
| `ISSUES.md` | 问题追踪 | 遇到问题时 |
| `DEVELOPMENT_LOG.md` | 技术决策 | 重要决策时 |
| `CHANGELOG.md` | 版本记录 | 发布时 |

## 🎯 继续开发指南

### 每次开始开发前

1. **读取当前状态**
   ```bash
   cat docs/SESSION_MEMORY.md
   ```

2. **检查待处理任务**
   ```bash
   grep "🔴\|🟡" docs/SESSION_MEMORY.md
   ```

3. **查看最近提交**
   ```bash
   git lg -10
   ```

4. **检查问题追踪**
   ```bash
   cat docs/ISSUES.md
   ```

### 每次开发后

1. **更新本文件**
   ```markdown
   ### 进行中的任务
   - [x] 已完成任务
   - [ ] 下一步任务
   ```

2. **记录问题（如有）**
   ```bash
   # 编辑 docs/ISSUES.md
   ```

3. **提交代码**
   ```bash
   git add -A
   git commit -m "描述"
   git push
   ```

## ⚠️ 注意事项

1. **每次开发前先 pull**
   ```bash
   git pull origin main
   ```

2. **提交前先格式化**
   ```bash
   npm run format
   ```

3. **遇到问题先查文档**
   - 查 `docs/ISSUES.md`
   - 查 `docs/DEVELOPMENT_LOG.md`

4. **重要决策要记录**
   - 修改架构
   - 更换技术方案
   - 发现的新问题
